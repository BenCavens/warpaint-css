Spacing

Warpaint provides margin and padding utility classes. the spacing of elements inside their parent container (margins) and spacing inside the element (paddings).

.stack

Handle vertical margins with .stack. Stack adds a top and bottom margin to an element. Give some vertical breathing room to page elements.

For grid columns, you can choose to apply gutters which will automatically space out the columns both horizontally as vertically.

.inline

Handle horizontal spacing with .inline. E.g. adding breathing room between an image and its paragraph on the right.

You can opt to use a .inline-group class on the parent element. This will effect its child elements to behave as if they are given the .inline style. Ideal for spacing elements in sequence like pills, tags, breadcrumbs or buttons.

.inset

Padding of an element can be managed by .inset, .stretched or .squished. Inset is the most common and sets a square padding (equal padding on four sides of the element).

.squished

A squished inset stretches horizontally. It reduces space top and bottom by 50% of the inset. This is mostly used for buttons and cell-like containers as tables or lists

.stretched

A stretched padding is the opposite of squished. It stretches vertically and is ideally for textboxes and other form elements

Responsive sizes

The same breakpoint prefixes can be applied to the spacing classes. E.g. .inset .l-inset-l will set standard inset but from the large breakpoint a large inset will be applied.

Gutters

If you like some spacing between columns, add the .gutter class to your .row element. This way all child columns will receive an equally spaced gutter between them.

I'm guttered
I'm guttered
We're all guttered
We're all guttered
We're all guttered
<div class="row gutter">
    <div class="column"><div class="box">I'm guttered</div></div>
    <div class="column"><div class="box">I'm guttered</div></div>
</div>

It is important to note that when using gutters, the column element acts as a wrapper of the displayed content. The gutter logic add padding to each column in order to realize flexible spacing in between.

Instead of styling the column itself you should rather apply styles on the inner child element.

Overflow due to negative margins

For optimal positioning of guttered columns, the parent row has negative margins on left and right sides. There is no issue when using the container wrapper but if you have columns from edge to edge of the viewport, be aware that the negative margins will cause an overflow on your body element.

Hide the horizontal overflow on the body like so: body{ overflow-x:hidden; }

Responsive sizing

Warpaint utilizes 5 breakpoints You can set a specific column size for each of these breakpoints.

Prepend the column class with a breakpoint prefix like xs-, s-, m-, l- or xl-. E.g. Adding .l-column-3 to your column will enforce a 1/4 column size for all viewports equal or wider than the large breakpoint (around 64em)

.l-column-3 .column-6
.l-column-3 .column-6
.l-column-3 .column-6
.l-column-3 .column-6

    <div class="l-column-3 column-6">...</div>
    <div class="l-column-3 column-6">...</div>
</div>

Multiple sizes

Sizing logic is calculated from smaller to larger viewports (aka mobile first). You can add responsive sizes for each breakpoint. However it is good practise to just set a default column size and only add responsive sizing if it's needed. 

<!-- Don't do this -->
<div class="row">
    <div class="xs-column-6 l-column-6 xl-column-3"></div>
    ...

<!-- instead do this -->
<div class="row">
    <div class="column-6 xl-column-3"></div>
    ...
