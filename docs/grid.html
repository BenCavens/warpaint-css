Getting started

Installation

Current stable version is 0.0.9.

Install warpaint via npm

$ npm install warpaint --save-dev

Or download from github:

Full zip warpaint.min.css (17 kb)

Browser support

Out of the box, warpaint is supported in all modern browsers: Chrome, Firefox, Safari, Edge and Opera.

There is a fairly good support for IE10+ and most functionality should be working. Since it relies on flexbox there is limited compatibility with IE9.

Customize build

You can build from the sass files yourself if you like to customize the warpaint logic.

  1. After installing the package via npm, copy the following file to your application: node_modules/warpaint/src/sass/settings.scss
  2. In your main sass file refer to this settings file as wel as to the warpaint compilations file. This is located at node_modules/warpaint/src/sass/warpaint.scss The warpaint file is the main file for compilation and contains all the source references.
  3. The settings file contains all the customisable features and you can change these to your liking. Note: in order to to override the default settings, be sure to import the settings source before importing the warpaint file.

Settings

Container margin

By default the container has an equal margin to the left and right. These margins are responsive depending on the viewport width. Here you can define the maximum value.

$container-margin: 16% !default;

Grid columns

By default the grid consists of 12 blocks but you can set this whatever you like. If you feel experimental, we suggest some good alternatives such as a 10 or 15 block division.

$grid-columns: 12 !default;

Base font size

The default font-size. This is also the value on which all font based spacings and sizings are measured upon. By default this is set to 0.875em. Depending on your font family of choice this should be changed to a proper default.

$base-font-size:  0.875em !default;

Size ratios

Based upon the base font size, here we set the font-size ratios for the different screen sizes. These values are carefully selected to allow for a responsive typography. Important since font-size determines the sizing of a lot of components

$ratio-xs: 0.8 !default;
    $ratio-s: 0.9 !default;
    $ratio-m: 1.0 !default;
    $ratio-l: 1.1 !default;
    $ratio-xl: 1.2 !default;
    $ratio-xxl: 1.4 !default;

Spacing

Based upon the base font size, These values determine the spacing of elements like margins and paddings. These settings are the base of all responsive sizings.

$space-xs: 0.375rem !default;
    $space-s: 0.75rem !default;
    $space-m: 1.5rem !default;
    $space-l: 3rem !default;
    $space-xl: 6rem !default;

Container

To present your content with some space to the left and right, place your rows inside a .container wrapper. By default the grid expands to the full width of its parent.

        <div class="row">
            <div class="column">This is content</div>

        </div>
    </div>

Gutters

If you like some spacing between columns, add the .gutter class to your .row element. This way all child columns will receive an equally spaced gutter between them.

I'm guttered
I'm guttered
We're all guttered
We're all guttered
We're all guttered
<div class="row gutter">
        <div class="column"><div class="box">I'm guttered</div></div>
        <div class="column"><div class="box">I'm guttered</div></div>
    </div>

It is important to note that when using gutters, the column element acts as a wrapper of the displayed content. The gutter logic add padding to each column in order to realize flexible spacing in between.

Instead of styling the column itself you should rather apply styles on the inner child element.

Overflow due to negative margins

For optimal positioning of guttered columns, the parent row has negative margins on left and right sides. There is no issue when using the container wrapper but if you have columns from edge to edge of the viewport, be aware that the negative margins will cause an overflow on your body element.

Hide the horizontal overflow on the body like so: body{ overflow-x:hidden; }

Responsive sizing

Warpaint utilizes 5 breakpoints You can set a specific column size for each of these breakpoints.

Prepend the column class with a breakpoint prefix like xs-, s-, m-, l- or xl-. E.g. Adding .l-column-3 to your column will enforce a 1/4 column size for all viewports equal or wider than the large breakpoint (around 64em)

.l-column-3 .column-6
.l-column-3 .column-6
.l-column-3 .column-6
.l-column-3 .column-6
Resize the viewport to view the change. Active breakpoint is . 
<div class="row">
        <div class="l-column-3 column-6">...</div>
        <div class="l-column-3 column-6">...</div>
        <div class="l-column-3 column-6">...</div>
        <div class="l-column-3 column-6">...</div>
    </div>

Multiple sizes

Sizing logic is calculated from smaller to larger viewports (aka mobile first). You can add responsive sizes for each breakpoint. However it is good practise to just set a default column size and only add responsive sizing if it's needed. 

<!-- Don't do this -->
    <div class="row">
        <div class="xs-column-6 l-column-6 xl-column-3"></div>
        ...

    <!-- instead do this -->
    <div class="row">
        <div class="column-6 xl-column-3"></div>
        ...
