Warpaint

A responsive typographic stylesheet
currently v0.0.2

Download v0.0.2 Get started

Quickstart

To get started, download v0.0.2 and include the warpaint.min.css in the head of your html: 

<link rel="stylesheet" href="/warpaint.min.css" type="text/css">

Next, read through the full documentation.

Warpaint includes normalize.css by default. Even with normalize included, Warpaint is small in size. Roughly 10.9879KB minified.

Note: If you build warpaint from sass, you can opt to leave normalize out.

Browser support

Out of the box, warpaint is supported in all modern browsers: Chrome, Firefox, Safari, Edge and Opera.

There is a fairly good support for IE10+ and most functionality should be working. Since it relies on flexbox there is limited compatibility with IE9. For those in need for IE9 support, we provide a separate stylesheet which you can load via a conditional comment: 

<link rel="stylesheet" href="/warpaint.min.css" type="text/css">
<!--[if IE 9]><link rel="stylesheet" href="/warpaint-ie9.css" type="text/css" /><![endif]-->

Custom build

You can customize Warpaint by including the main sassfile src/sass/warpaint.scss into your own build process. You can then edit the src/sass/settings.scss file which contains all the customisable features. The most important settings are: 

// Grid columns
$grid-columns: 12

// Base fontsize - root for all spacing units
$base-font-size:  0.875em

Why Warpaint?

Warpaint is a responsive typographic stylesheet. It is a collection of classes to help you maintain consistent spacing and typography throughout your entire css.

Its focus lies on:

  • Focus on creating beautiful responsive site, not a 'pixel-perfect' design replication.
  • Use of relative units for consistency of all your spacing and margins
  • Globalized spacing: maintainting harmony of the typographic elements
  • Avoid conflicts with classbased approach

Mindset

Warpaint is not so much intended for jumpstarting the website UI development. It's not a full blown frontend framework for blasting out UI prototypes. You have the well-known bootstrap or foundation projects for this purposes. Warpaint doesn't push your frontend work to get any faster. Faster development leads to more monotone, generic designs. Warpaint tries to assist you in your coding process, and not so much short circuit your visual creativity. It does however give constraints on spacing and sizing.

Relative units

Never set font-size to a pixel value because it overrides the user’s browser settings. Use em or rem instead.

  • It’s a best practice to use relative units like em for font-size.
  • Size in em if the property scales according to it’s font-size
  • Size everything else in rem, like margin or padding. If it is more parent-focused rather than content-focused, you should use %. e.g. .container
  • Increase font-size and line-height as screen sizes increase.

Globalized spacing

Spacing and sizing values are set globally for the entire project. Never set an unique padding or margin value to an element. This lead to disharmony of your page. E.g. setting .button{ padding:15px; } is something Warpaint tries to avoid.

  • focus on maintaining and respecting the spacings and element boundaries. Your whitespace is space too!
  • reduce risk of these isolated measurements 'margin and padding creep'. (e.g. setting your own random margin of 10px on a button)
  • Honors the vertical rhythm

Class based approach

  • a set of predefined classes that each serves a single purpose.
  • The classnames reflect what they represent.
  • It helps to avoid cascade hierarchy conflicts. These usually lead to a messy nesting of selectors and ultimately you are forced to pull the dreaded !important card. By defining css with single-purpose classes we keep a low specificity, avoid css property wars and keep the codebase open for extension.
